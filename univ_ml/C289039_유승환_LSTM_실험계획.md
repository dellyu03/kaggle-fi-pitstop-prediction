c# LSTM 실험 계획서 — F1 Pit Stop 예측 (범주형 임베딩 포함)

> Kaggle Playground Series S6E5 · 타겟 `PitNextLap` (이진 분류) · 평가지표 ROC-AUC
> 과거 N랩 시퀀스로 **다음 랩 피트 여부**를 예측한다.
> 수업 LSTM(수치형 전용)을 확장하여 **범주형(Driver/Compound/Race)을 Embedding으로 시퀀스에 결합**한다.

---

## 0. 실험 목적

1. 과거 랩 시퀀스의 시계열 패턴으로 피트 타이밍을 예측할 수 있는가? (OOF AUC)
2. 범주형 임베딩(드라이버 성향·타이어·서킷)이 수치형 전용 대비 성능을 올리는가?
3. 과거 몇 랩(window_size)을 봐야 가장 잘 맞히는가? (3 / 5 / 10)

---

## 1. 데이터 & 피처

| 구분 | 피처 | 처리 |
|------|------|------|
| 연속형(8) | `LapNumber`, `TyreLife`, `LapTime (s)`, `LapTime_Delta`, `Cumulative_Degradation`, `RaceProgress`, `Position`, `Position_Change` | StandardScaler |
| 범주형(3) | `Driver`(≈887), `Compound`(5), `Race`(26) | LabelEncoder → Embedding |
| 타깃 | `PitNextLap` (0/1, pos rate ≈ 0.199) | `class_weight` |
| 그룹 키 | `Driver`, `Race`, `Year` (시퀀스 단위) | GroupKFold |

---

## 2. 전체 파이프라인 (스텝 요약)

```
Step1 정렬 → Step2 범주형 인코딩 → Step3 연속형 스케일링(fold 내부)
   → Step4 슬라이딩 윈도우 → Step5 GroupKFold 분할
   → Step6 모델(임베딩+LSTM) → Step7 학습설정 → Step8 HPO
   → Step9 OOF 평가 → Step10 window 민감도 → Step11 지표 → Step12 제출
```

---

## Step 1 — 데이터 정렬

- **의도**: 슬라이딩 윈도우는 "과거→현재" 순서를 전제로 한다. 행 순서가 섞여 있으면 시퀀스가 시간 역행하거나 다른 드라이버의 랩이 섞일 수 있다.
- **기술**: `sort_values(['Driver','Race','Year','LapNumber'])` 로 그룹별 랩을 시간순 정렬 후 `reset_index`.
- **결과**: 각 (Driver, Race, Year) 블록이 LapNumber 오름차순으로 연속 배치.

---

## Step 2 — 범주형 인코딩 (LabelEncoder)

- **의도**: Driver/Compound/Race를 신경망이 받을 수 있는 **정수 코드**로 변환. 임베딩 층의 입력은 정수 인덱스여야 한다.
- **기술**: `LabelEncoder`. **train+test를 합쳐 fit** → test에만 있는 카테고리(unseen)로 인한 에러 방지. 라벨 번호만 부여하므로 누수가 아니다.
- **세부**: 각 피처의 고윳값 수(`cat_dims`)를 임베딩 `input_dim`으로 사용.

---

## Step 3 — 연속형 스케일링 (StandardScaler)

- **의도**: `LapTime (s)`(수십~수백)와 `Position_Change`(±소수)처럼 **스케일 차이가 크면** LSTM 학습이 불안정/발산. 평균0·분산1로 맞춰 안정화.
- **기술**: `StandardScaler`. **반드시 fold의 train 부분으로만 fit** 후 val/test에 `transform` → 검증 누수 방지.
- **주의**: 범주형 코드는 스케일링하지 않는다(임베딩이 처리).

---

## Step 4 — 슬라이딩 윈도우 구성

- **의도**: 각 샘플이 **과거 정보만** 담도록 하여 한 샘플 내부의 미래 누수를 구조적으로 차단. "과거 N랩 → 다음 랩 피트" 형태로 시계열 학습.
- **기술**: 그룹별로 정렬 후 `[i-window_size : i]` 를 입력, `i` 시점 타깃을 라벨로 수집.
  - 출력 3종: `Xn`(N, ws, 8 연속형), `Xc`(N, ws, 3 범주형 코드), `y`(N,), `groups`(N, Race+Year).
- **세부**: window보다 짧은 레이스는 샘플이 0개 → 자동 제외.

```
[랩1..랩5] → 랩6 PitNextLap
[랩2..랩6] → 랩7 PitNextLap   (window_size=5 예시)
```

---

## Step 5 — 데이터 분할 (GroupKFold)

- **의도**: 같은 레이스의 랩이 train/val에 동시에 들어가면 "이미 본 레이스"를 맞히는 셈 → **점수 부풀림(누수)**. 레이스 단위로 분리해 공정 평가.
- **기술**: `GroupKFold(n_splits=5)`, `groups = Race+Year`. 같은 그룹은 한 fold에만 배치.
- **검증**: train/val 간 겹치는 레이스 수 = 0 확인.

---

## Step 6 — 모델 (범주형 Embedding + LSTM)

- **의도**: 드라이버 성향·타이어·서킷 정보를 시퀀스에 결합. 범주형을 원-핫이 아닌 **밀집 임베딩**으로 학습해 차원·표현력 효율화.
- **기술**: Keras **함수형 API**(다중 입력). 각 타임스텝 범주형 코드 → `Embedding` → 시퀀스 임베딩 → 연속형과 마지막 축으로 `Concatenate` → `LSTM`.
  - 임베딩 차원: Driver=16, Compound=3, Race=6.
- **구조**:
```
[연속형(ws,8)] + [Driver_emb(ws,16), Compound_emb(ws,3), Race_emb(ws,6)]
   → Concatenate → LSTM(128, return_sequences=True) → LSTM(128)
   → Dropout(0.1) → Dense(1, sigmoid)
```
- **분류 전환(수업 회귀 대비)**: 출력 `sigmoid`, loss `binary_crossentropy`, metric `AUC`.

---

## Step 7 — 학습 설정

- **의도**: 불균형 대응 + 과적합 억제 + 적절한 시점 학습 종료.
- **기술**:
  | 기법 | 설정 | 의도 |
  |------|------|------|
  | `class_weight={0:1, 1:4.03}` | pos_weight=(1-p)/p | 소수 클래스(피트) 가중 |
  | `Dropout(0.1)` | LSTM 뒤 | 과적합 억제 (수업과 동일) |
  | `EarlyStopping(val_loss, patience=epoch*0.3)` | best 가중치 복원 | 과학습 방지 (수업과 동일) |
  | `Adam(1e-3)` | — | 수업과 동일 |
  | seed 42 | numpy/tf | 재현성 |

---

## Step 8 — HPO ({epochs, batch_size} grid search)

- **의도**: 수업에서 다룬 하이퍼파라미터 탐색을 적용해 epoch/batch 조합 최적화.
- **기술**: 수업은 `scikeras + GridSearchCV`를 썼으나 현재 환경의 `scikeras 0.13`이 `sklearn 1.8`과 비호환 → **동일 탐색공간을 수동 grid search**로 구현(GroupKFold 2-fold + ROC-AUC).
  - 탐색공간: `epochs ∈ {20,40}`, `batch_size ∈ {16,32}` (수업과 동일).
  - 계산량 절감을 위해 표본 4만으로 베스트 조합 선택.

---

## Step 9 — OOF 평가 (최종 성능)

- **의도**: 누수 없는 **out-of-fold** 예측으로 일반화 성능을 정직하게 측정. 다른 모델(GBDT 등)과 같은 기준 비교.
- **기술**: 베스트 `{epochs, batch_size}`로 GroupKFold 5-fold 루프 → 각 fold val 예측을 `oof`에 누적 → `roc_auc_score(y, oof)`.
- **기록**: fold별 AUC, 평균, 표준편차(안정성).

---

## Step 10 — window_size 민감도 분석

- **의도**: "피트 징후가 몇 랩 전부터 나타나는가"라는 도메인 질문을 데이터로 검증. 최적 시야 결정.
- **기술**: `window_size ∈ {3,5,10}` 각각 윈도우 재구성 → 동일 OOF 절차 → AUC 비교.
- **유의**: window=10은 짧은 레이스 표본이 줄어듦(주의).

---

## Step 11 — 평가 지표 / 시각화

- **의도**: 단일 AUC 외에 임계값 기반 실제 분류 품질, 오류 유형까지 확인.
- **기술**:
  - ROC-AUC(주), F1(thr=0.5), Confusion Matrix.
  - ROC Curve 시각화 (`matplotlib`).

---

## Step 12 — 전체 재학습 → Test 추론 → 제출

- **의도**: CV로 검증한 설정으로 전체 데이터를 학습해 최종 모델 확보, test 전 행에 예측 부여.
- **기술**:
  - 전체 train으로 재학습(best epoch/batch) → `best_lstm.keras` 저장.
  - test 윈도우 생성: 초반 랩(과거 < window)은 첫 랩으로 **left-pad**하여 모든 id에 예측.
  - 예측 확률을 `id, PitNextLap`로 정렬·저장 → `submission_lstm.csv`.

## 3. 가설

| 가설 | 근거 |
|------|------|
| window=5 부근 최적 | 피트 결정은 통상 3~5랩 전 타이어 열화 징후로 판단 |
| 범주형 임베딩 추가 시 AUC 향상 | 드라이버 성향·타이어(Compound는 스틴트마다 변함) 신호 추가 |
| 스케일링이 학습 안정화·성능 기여 | 연속형 스케일 차이 큼 |
| EarlyStopping이 fold 분산 감소 | 과적합 억제 |

## 5. 리스크 & 대응

| 리스크 | 대응 |
|--------|------|
| window보다 짧은 레이스 그룹 | 자동 제외(window=10에서 표본 급감 주의) |
| 범주형 unseen 카테고리 | LabelEncoder를 train+test 합쳐 fit |
| 스케일링 누수 | scaler는 train fold로만 fit |
| 불균형으로 1을 잘 못 맞힘 | `class_weight`, F1·confusion matrix 병행 |
| 임베딩 과적합(Driver 887종) | dropout↑, 임베딩 차원↓, EarlyStopping |
| 학습 불안정/발산 | lr↓(5e-4), batch↑, 스케일링 |
| 연산량 과다(전체+5fold+민감도) | HPO 표본 축소, 민감도는 후순위 실행 |
